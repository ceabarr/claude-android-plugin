# Privacy Policy

Claude Android QA Plugin is a local Claude Code plugin for Android emulator QA.

## Data Collection

This plugin does not collect, transmit, sell, or share personal data.

The plugin does not include telemetry, analytics, tracking, advertising, or a hosted backend service.

## Local Data Processed

When used, the plugin may help Claude Code run local commands that interact with Android Debug Bridge (`adb`) and Android emulators or connected devices.

Depending on how you use it, local outputs may include:

- screenshots
- `uiautomator` XML
- compact UI summaries
- logcat output
- package and activity names
- emulator or device serials
- command output

These files are created locally in your project or working directory when requested. They are not sent to me by this plugin.

## Sensitive Information

Android screenshots, UI XML, and logcat output can contain sensitive information from your app or device.

Review generated files before sharing them publicly or attaching them to issues, discussions, pull requests, or support requests.

Do not share:

- API keys
- access tokens
- private app source code
- customer or user data
- sensitive logs
- screenshots containing private information

## Third Parties

This plugin depends on tools you run locally, such as Claude Code, Python, `adb`, Android emulators, connected Android devices, and Gradle. Those tools may have their own privacy and security behavior.

This plugin does not control third-party tools or services.

## Contact

For privacy or security questions, use GitHub Issues for general questions. For security-sensitive reports, use GitHub private vulnerability reporting for this repository.

Repository: https://github.com/ceabarr/claude-android-plugin

## Changes

This policy may be updated as the plugin changes. Material changes will be reflected in this file.
